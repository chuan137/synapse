# Critic Patches

This directory contains proposed protocol patches generated by the critic (`synapse eval --critic`).

## Frontmatter format

Each patch file begins with YAML frontmatter:

```
---
target_file: templates/SYNAPSE-developer.md
target_role: developer
failure_metric: tool_calls
---
```

| Field | Values | Meaning |
|---|---|---|
| `target_file` | `templates/SYNAPSE-worker.md`, `templates/SYNAPSE-orchestrator.md`, `templates/SYNAPSE.md` | Which protocol file the patch targets |
| `target_role` | role name or `null` | The agent role that failed; `null` = cross-role patch |
| `failure_metric` | `tool_calls`, `has_commit`, `traceability_score`, `active_duration_ms` | The metric that triggered the critic |

## Workflow

1. `synapse eval --critic` generates patches here for failed trajectories.
2. `synapse eval --gate` validates each patch (regression + coverage verdict).
3. `synapse eval-apply <patch-file>` applies a gate-approved patch.
4. Operator reviews before any SYNAPSE*.md files are modified.
